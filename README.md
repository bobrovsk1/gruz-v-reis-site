# Груз в Рейс

Десктопный прототип веб-приложения для грузоперевозок с локальным JSON backend. Интерфейс сделан как рабочий кабинет: боковое меню, расчет рейса, активные заявки, карта маршрута, вход и регистрация по телефону.

## Что внутри

- `index.html` - разметка десктопного приложения.
- `styles.css` - визуальный стиль, похожий на продуктовый логистический кабинет.
- `script.js` - клиентская логика кабинета, загрузка данных из API, модальное окно входа, регистрация, SMS/Telegram demo-подтверждение.
- `languages.js` - словарь интерфейса для русского и английского языка.
- `request.html` - отдельная вкладка с шаблоном размещения заявки.
- `request.js` - отправка новой заявки в backend с fallback-сохранением черновика в браузере.
- `assets/route-board.svg` - локальный визуальный ассет маршрута.
- `server.js` - локальный сервер без зависимостей: статика, JSON API, хранение данных в `data/*.json`.
- `data/` - JSON-хранилище заявок, перевозчиков, документов, пользователей, OTP-кодов и сессий. Папка создается при первом запуске.

## Backend API

- `GET /api/health` - проверка работоспособности backend.
- `GET /api/orders` - список заявок.
- `POST /api/orders` - создание заявки из формы `request.html`.
- `GET /api/carriers` - список перевозчиков.
- `GET /api/documents` - список документов.
- `POST /api/documents` - сохранение метаданных выбранных документов.
- `POST /api/auth/request-code` - запрос demo-кода подтверждения.
- `POST /api/auth/verify` - проверка demo-кода и выдача session token.
- `GET /api/profile` / `PATCH /api/profile` - чтение и обновление профиля, компании, ИНН и пользователей компании по Bearer token.

## Авторизация

Сейчас реализован демонстрационный backend-сценарий:

- вход по номеру телефона;
- регистрация по номеру, имени, роли и компании;
- выбор канала подтверждения: SMS или Telegram;
- одноразовый demo-код `2486` или значение из `DEMO_OTP`;
- сохранение подтвержденного профиля на сервере и session token в `localStorage`.

Для настоящей отправки кодов нужно подключить SMS-провайдера и Telegram Bot API в обработчике `POST /api/auth/request-code`.

## Запуск

Для локального сервера и API:

```bash
node server.js
```

Адрес по умолчанию: `http://localhost:4173`.

Отдельная ссылка на админскую панель: `http://localhost:4173/admin.html`.
В основном интерфейсе вход в админку не показывается.

Настройки через переменные окружения:

- `PORT` - порт сервера, по умолчанию `4173`.
- `DATA_DIR` - путь к JSON-хранилищу, по умолчанию `data/`.
- `DEMO_OTP` - demo-код подтверждения, по умолчанию `2486`.

Проверка синтаксиса:

```bash
npm run check
```
